c++与c的区别

兼容性

c是c++的子集，c++兼容c，同时有很多新特性，auto变量、nullptr、引用、智能指针等。

面对对象和面对过程

c++是面向对象的编程语言，c是面向过程的编程语言。

安全性

c有一些不安全的语言特性：指针使用的潜在危险？、强制转换的不确定性、内存泄漏。
c++增加了新特性来改善，如const常量、引用、cast转换、智能指针、try——catch等。

可复用性

c++的可复用性高，引入了模板的概念，在此基础上，实现了方便开发的标准模板库STL，STL相对c的函数库更灵活更通用。

c++中class与struct的区别

1.struct一般用于描述一个数据结构的集合，而class是一个对象数据的封装。
2.struct中默认的访问控制权限？是public的，而class的默认访问权限是private的。
3.继承关系中，struct是共有继承，class是私有继承。
4.class关键字可以用于定义模板参数，就像typename，struct不能用于定义模板参数？。
ps：c++的struct对c中的进行了扩充，可以有成员函数、静态成员、访问控制可以修改、可以继承、可以初始化。使用时可以省略struct关键字。

include头文件的顺序和双引号””和尖括号<>的区别

区别

1.<>的头文件是系统文件，双引号””的头文件是自定义文件。
2.编译器预处理阶段查找头文件的路径不一样。

查找路径

1.使用尖括号的头文件的查找路径：编译器设置的头文件路径–>系统变量。
2.双引号的：当前头文件目录–>编译器设置的头文件路径–>系统变量。

导入c函数的关键字

c++中导入c函数的关键字是extern，表达形式为**extern “C”**，加关键字的作用是为了能够正确实现c++代码调用其他c语言代码。加入关键字后，编译器这部分代码会按照c语言的编译进行编译，而非c++。

c++和c编译的区别

c++支持函数重载，编译器编译函数的过程中会将函数的参数类型加到编译后的代码中，而不仅仅只是函数名。
c不支持函数重载，编译时不会带上函数的参数类型，一般只包含函数名。

c++从代码到可执行二进制文件的过程

共四个过程，预编译、编译、汇编、链接。

预编译

1.将所有的#define删除，展开所有宏定义；
2.处理所有的条件预编译指令如：#if，#ifdef；
3.处理所有的#include预编译指令，将被包含的文件插入到该预编译指令的位置；
4.过滤所有注释；
5.添加行号和文件名标识。

编译

1.词法分析：将源代码的字符序列分割成一系列记号；
2.语法分析：对记号进行语法分析，产生语法树；
3.语义分析：判断表达式是否有意义；
4.代码优化：
5.目标代码生成：生成汇编代码；
6.目标代码优化。

汇编

将汇编代码转变成机器可执行的指令。

链接

将不同的源文件产生的目标文件进行链接，形成一个可执行的程序。分静态和动态两种链接。
【静态链接】：在链接的时候就已经把要调用的函数代码或过程链接到生成的可执行文件中，就算把静态库删除也不影戏可执行程序的执行；生成的静态链接库，win下以.lib为后缀，linux下以.a为后缀。
【动态链接】：链接的时候没有把调用的函数代码链接进去，在执行过程中再去找链接的函数，生成的可执行文件中没有函数代码，只包含函数的重定位信息，所以删除动态库时，可执行程序不能运行。生成的动态链接库，win下以.ddl为后缀，linux下以.so为后缀。

static关键字的作用

1.定义全局静态变量和局部静态变量

变量前加static关键字，初始化的静态变量会在数据段分配内存，未初始化的静态变量在BSS段（Block Started by Symbol）分配内存。直到程序结束，静态变量始终维持前值，全局静态变量和局部静态变量的作用域不一样。

2.定义静态函数

函数返回类型前加static，函数被定义为静态函数。静态函数只能在本源文件？中使用。

3.定义类中的静态成员变量和静态成员函数

类中的静态数据成员拥有单独的存储区，创建的该类的所有对象，这些对象的静态数据成员共享这块静态存储空间。
ps：调用一个对象的非静态成员函数时，系统会把该对象的起始地址赋给成员函数的this指针。但静态成员函数不属于任何一个对象，c++规定静态成员函数没有this指针。没有this指针，没有指向某一对象，也就无法对一个对象中的非静态成员进行访问。

实习准备的开始

c++语言的特点

1.是面向对象的，兼容C。
面向对象就是把现实问题抽象为对象，通过调用每个对象的属性或功能去解决问题。
2.面对对象：三个特性，封装、继承、多态。
3.可读性好。
4.c++生成的代码质量高、运行效率高。
5.c++更安全，增加了const常量、引用、四类cast转换（static_cast、dynamic_cast、const_cast、reinterpret_cast）、智能指针、try——catch等等；
c语言可以使用强制类型转换进行变量间的类型转换，如int->double。c++保留了这种转换，并增加了类之间的转换，一共有四种：
- static_cast：static_cast < type-id > ( expression )
基本数据类型之间的转换，nonconst->const（不可逆转）；
空指针->目标类型指针；
任何类型的表达式->void类型；
子类类型的指针->父类类型指针 upercasting（可以逆转 downcasting，安全性由开发人员保证）;
【C++禁止void*隐式转换为其他类型指针，反之可以。C语言二者都允许】
-dynamic_cast：
使用了RTTI来做类型检查，父类转子类的时候使用它更安全。如果是不能转换的类型，转引用会抛std::bad_cast异常，转指针会返回nullptr。
用得非常少，因为一用就会引入RTTI，造成代码膨胀。而且需要根据实际类型作出行动可以抽象成虚函数，不需要cast。
-const_cast:
修改类型的const或volatile属性，除了const 或volatile修饰之外，type_id和expression的类型是一样的，一般用于强制消除对象的常量(const)性。常量指针、引用、对象被转化成非常量，并且仍然指向原来的对象。
可以利用它来改const变量的值，但是修改const变量是未定义行为，不要这么做。
eg：我们很早之前写了一个函数，是传入非const成员变量的，涉及到关联关系函数已经无法更改，但同时又新建了一个const变量要使用这个函数，就要使用const_cast了。（可以理解为把这个新的const变量的const属性消除掉？）
【const和volatile和RITTI下次再写吧T T】
-reinterpret_cast:
是在二进制层面上将值按照新类型重新解释，比方说查看指针地址的整数值就用
cout << reinterpret_caststd::uintptr_t(some_ptr)，
又或者想直接操纵浮点数的二进制就用【看不懂了】
reinterpret_caststd::int32_t&(some_float)。非常危险，少用

6.c++可复用性高。引入了模板的概念，在此基础上实现了方便开发的标准模板库STL，standard template library。
7.c++具有发展性。后续版本加入了更多新的特性，如c++11中引入了nullptr、auto变量，Lambda匿名函数、右值引用、智能指针。
